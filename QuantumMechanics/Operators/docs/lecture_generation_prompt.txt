PURPOSE
-------
This prompt template is used to generate a complete, fully-specified lecture
for any module in the QM/QC programme. Fill in the [BRACKETED] fields before
submitting. All output should be consistent with the programme's three-level
hierarchy (Series > Module > Lecture), five-tier pedagogy, and seven-artifact
assessment system as defined in the Pedagogical Support Material document.

GENERATE THE FOLLOWING SECTIONS
--------------------------------

1. LECTURE HEADER (metadata block)
   - Series / Module / Lecture codes and titles
   - Duration, target tiers, prerequisites
   - Summary (2–3 sentences): what this lecture covers and why it matters

2. LEARNING OBJECTIVES (per tier)
   For each active tier, provide 3–5 bullet-point objectives in the form:
   "By the end of this lecture, a [TIER] student should be able to..."
   Use action verbs calibrated to tier:
     HS     → identify, describe, compute (simple), interpret
     BegUG  → apply, calculate, expand, translate
     AdvUG  → prove, derive, construct, verify
     MSc    → analyse, generalise, connect, reformulate
     PhD    → formalise, extend, critique, classify rigorously

3. 90-MINUTE PACING PLAN
   Divide into 4–6 time blocks (e.g. 0–15 min, 15–45 min, etc.)
   For each block state:
     - Time range
     - Topic / activity
     - Tier-specific depth note (brief)
     - Transition sentence to next block

4. SECTION-BY-SECTION CONTENT (lecture body)
   Number sections as [LECTURE_CODE].[SECTION_NUMBER]
   e.g. L05.1, L05.2, L05.3 ...
   For each section:
     a) Statement of the main result or concept
     b) Key definitions (with notation consistent with programme conventions)
     c) Core derivation or argument (tier AdvUG baseline; mark PhD extensions)
     d) Physical/geometric interpretation
     e) Worked example (one per section minimum)
     f) Tier differentiation note (what HS/BegUG see vs. MSc/PhD extensions)

   Programme notation conventions (always use these):
     - Operators        : hat notation  \hat{A}, \hat{H}, \hat{U}
     - States           : Dirac ket/bra  |\psi\rangle, \langle\phi|
     - Density operator : \hat{\rho}
     - Commutator       : [\hat{A},\hat{B}] = \hat{A}\hat{B} - \hat{B}\hat{A}
     - Anti-commutator  : \{\hat{A},\hat{B}\} = \hat{A}\hat{B} + \hat{B}\hat{A}
     - Identity         : \hat{\mathbb{1}}  (or \hat{1})
     - Inner product    : \langle\phi|\psi\rangle
     - Partial trace    : \operatorname{Tr}_B

5. CONNECTIONS
   - Backward: explicit link to preceding lecture(s) by code + concept
   - Forward:  explicit link to upcoming lecture(s) by code + concept
   - Cross-module: any relevant connection to other Series II / III modules

6. ASSESSMENT ARTIFACT BUNDLE
   Generate one complete artifact for each of the following types.
   Each artifact must be consistent with the target tiers specified above.

   [A] CONCEPT QUESTIONS (CQ) — 8 questions
       Mix: multiple choice (3), true/false + justify (3), short answer (2)
       10–15 minutes. Cover all main concepts of the lecture.

   [B] SIMPLE PROBLEM SET (SPS) — 10 problems
       Structure: Part A (mechanics, 4 problems), Part B (interpretation, 3),
       Part C (translation, 3). Each 1–5 steps. Include answer key.

   [C] COMPLEX PROBLEM SET (CPS) — 4 problems
       Multi-part (a)(b)(c). Proof/derivation based. 3–6 hours total work.
       Include: problem statement layer only (hints in separate file).

   [D] SIMPLE PROJECT (SPJ) — 1 project
       2–6 hours. Structured. Deliverable: code/worksheet + 1–3 page report.
       Include: full spec with inputs, outputs, required checks, rubric.

   [E] COMPLEX PROJECT (CPJ) — 1 project
       8–15 hours. Open-ended. Deliverable: 5–8 page report + appendix.
       Include: full spec, analysis questions, rubric.

   [F] WELL-DEFINED RESEARCH QUESTION (WRQ) — 1 question
       5–10 hours. Answerable with literature + calculation.
       Include: question, method guide, deliverable format, rubric.

   [G] OPEN-ENDED RESEARCH QUESTION (ORQ) — 1 question
       10–20 hours. No single correct answer.
       Include: question, framing notes, deliverable format, rubric.

   Rubric weights for all artifacts:
     CQ  → 2 pts each; total 16 pts
     SPS → Correct result 60%, Method/notation 30%, Interpretation 10%
     CPS → Assumptions 15%, Proof structure 45%, Algebra 25%, Insight 15%
     SPJ → Correctness 40%, Tests 25%, Clarity 25%, Reproducibility 10%
     CPJ → Technical 30%, Synthesis 35%, Depth 25%, Originality 10%
     WRQ → Scope/method 25%, Evidence 25%, Technical 25%, Argument 25%
     ORQ → Clarity 25%, Depth 25%, Novelty 25%, Limitations 25%

7. EXERCISE PACKAGING (four-layer format)
   For the two hardest CPS problems and the SPJ, provide all four layers:
     Layer 1: Problem Statement (no hints)
     Layer 2: Suggestions / Hints (scaffolded)
     Layer 3: Advanced Suggestions (structural hints, alternative approaches)
     Layer 4: Detailed Solution (full derivation + commentary + pitfalls)

8. READING AND RESOURCES
   - Primary textbook references (chapter/section where possible)
   - Recommended: Dirac (Principles), Sakurai & Napolitano (Modern QM),
     and any module-specific texts from Appendix B of the syllabus
   - One historical source if appropriate
   - One research article for MSc/PhD tier (with brief annotation)

USAGE NOTES
-----------

1. MINIMAL VERSION (quick generation)
   If you only need the lecture body and CQ + SPS (no full artifact bundle),
   replace section 6 with:
     "Generate only: [A] CQ (8 questions) and [B] SPS (10 problems)."
   and omit sections 7 and 8.

2. ASSESSMENT-ONLY MODE
   To generate just the artifact bundle for an existing lecture, replace
   sections 1–5 with:
     "The lecture content is provided below. Generate only sections 6–8."
   Then paste the existing lecture body.

3. TIER-SPECIFIC VERSION
   To generate a version for a single tier only, add:
     "Generate all content exclusively for Tier [N] ([TIER_LABEL]).
      Do not include multi-tier differentiation tables."

4. SERIES III (Quantum Chemistry) ADAPTATION
   For Series III modules, additionally include:
     "Use [chembox] tcolorbox style instead of [modulebox].
      Replace programme colour scheme: chemgreen / lightgreen."
   And add to notation conventions:
     - Molecular Hamiltonian: \hat{H} = \hat{T}_n + \hat{T}_e + \hat{V}_{ne}
                               + \hat{V}_{ee} + \hat{V}_{nn}
     - Basis functions: Slater-type (STO) or Gaussian-type (GTO) as appropriate
     - Density functional: E_{xc}[\rho]

5. CHAINING LECTURES
   To generate a sequence of lectures consistently, add to the prompt:
     "This is lecture [N] of [TOTAL] in module [MODULE_CODE].
      The following lectures have already been generated: [LIST].
      Ensure all cross-references and forward connections are consistent."
